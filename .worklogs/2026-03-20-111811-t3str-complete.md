# t3str: Complete CLI for test discovery and execution

## Summary

Built `t3str`, a Rust CLI tool that replaces Python script generation and `extract-tools` for test discovery and execution across 9 languages. Also integrated t3str into the websmasher dev-process-v3 pipeline (separate commit in websmasher repo).

## What Was Done

### Phase 1: Scaffold
- Hexagonal architecture workspace at `apps/t3str/` with 6 crates
- Domain types: Language, TestFile, TestResult, TestSuite, TestSummary, T3strError
- Ports: TestDiscoverer and TestExecutor traits
- Commands: ExtractCommand and RunCommand use cases
- CLI: clap-based with `extract` and `run` subcommands, `--format json|human`

### Phase 2: Run Adapter (49 tests)
- 9 language executors spawning native test runners (pytest, go test, cargo test, dotnet test, phpunit, rspec, jest, mvn/gradle, mix test)
- Structured output parsers: JUnit XML, Go JSON, cargo text, RSpec JSON, Jest JSON, dotnet stdout, mocha text, Nette text, mix text
- Isolated environments (GOPATH, CARGO_HOME, .venv detection)

### Phase 3: Extract Adapter (38 tests)
- 9 tree-sitter discoverers for AST-based test function detection
- Shared helpers: find_test_files, extract_with_query, discover_in_files
- Language-specific queries: #[test] (Rust), def test_* (Python), func Test* (Go), it()/test() (JS), @Test (Java/C#), it "desc" (Ruby), test "desc" (Elixir), function test* (PHP)

### Phase 4: Pipeline Integration (websmasher repo)
- s06_run_parsers.rs: Replaced 525-line Python script generator with 145-line t3str run call
- s04_clone_and_install.rs: Replaced extract-tools with t3str extract
- Removed tempfile dependency (no longer needed)

## Key Decisions

- **tree-sitter 0.24 API**: Uses `LANGUAGE` constants + `StreamingIterator` for query matches
- **tree-sitter-php 0.23.5**: Downgraded from 0.24.2 for compatibility (language version 14 vs 15)
- **No regex crate**: Banned in deny.toml; all parsing via tree-sitter queries or string methods
- **Output format**: t3str uses its own structured format (TestSuite with results/summary), not the old pipeline format

## Key Files

- `apps/t3str/Cargo.toml` — workspace root
- `apps/t3str/crates/adapters/outbound/extract/src/helpers.rs` — shared tree-sitter infrastructure
- `apps/t3str/crates/adapters/outbound/run/src/helpers.rs` — shared execution infrastructure
- `apps/t3str/crates/adapters/inbound/cli/src/main.rs` — CLI composition root
- `apps/t3str/CLAUDE.md` — architecture guide for future sessions
