# Migrate shedul3r HTTP adapter: Axum → actix-web, rename api → rest

**Date:** 2026-03-16 14:09
**Scope:** apps/shedul3r/ (inbound adapter crate renamed and migrated)

## Summary

Renamed `crates/adapters/inbound/api` to `rest` and migrated from Axum to actix-web 4. All 41 workspace tests pass. Prerequisite for adding MCP transport.

## Context & Problem

Need MCP transport for shedul3r. Official Rust MCP SDK (rmcp) has actix-web transport. Migrate existing HTTP adapter to actix-web to avoid mixing two frameworks.

## Decisions Made

### actix-web 4 over keeping Axum
- **Chose:** Full migration to actix-web
- **Why:** rmcp-actix-web provides official MCP Streamable HTTP transport. One framework is cleaner than two.

### Auth middleware as Transform/Service
- **Chose:** Full actix-web middleware implementation
- **Why:** Idiomatic pattern, cleaner than wrap_fn closures
