## Bugs to fix next session

### 1. Circuit breaker TTL not deployed
shedul3r 0.5.1 released on GitHub but the claude-worker image was built before the release. Need to rebuild the worker to pick up 0.5.1. Empty commit + push will trigger rebuild.

### 2. Step 6 structured output parsers returning 0 tests
The structured output files (test-output.json, test-output.xml) are written to the clone_dir on the Railway volume, but the parser functions look for them in the clone_dir. This should work but many parsers return 0. Likely cause: the test command writes output to the clone dir, but the Python script runs in a different working directory (the uploaded temp dir). The `os.path.join(clone_dir, "test-output.xml")` path is correct but the test command's `--junitxml=test-output.xml` writes relative to cwd, not to clone_dir.

**Fix:** Change test commands to use absolute paths: `--junitxml={clone_dir}/test-output.xml` instead of `--junitxml=test-output.xml`.

### 3. Step 4 install: 17/21
4 libraries fail to install:
- eikendev/sectxt (Rust): aws-lc-rs needs cmake (now in Dockerfile, untested)
- movitz-s/security-txt-node-parser (JS): needs TypeScript (`tsc` not found)
- joker314/security.txt-extension (JS): browser extension, no package.json
- One more (varies by run)

**Fix:** Add `typescript` to npm install for JS libs. Accept that some libs (browser extensions) can't be installed.

### 4. Testing efficiency
Each step 6 iteration takes ~5 minutes (21 remote commands). Debugging one Python error required 5+ iterations. Need a way to test the generated Python script locally before sending it to the remote worker.

**Fix:** Add a `--dry-run-script` flag that writes the generated test script to a local file for inspection. Or better: generate the script, compile-check it locally with `python3 -c "compile(open('script.py').read(), 'script.py', 'exec')"` before uploading.

## Key architectural decisions made

1. **Everything runs on Railway, nothing local** — all script steps use `run_remote_command`
2. **Dockerfile over volume installs** — runtimes baked into container image, not installed at runtime
3. **Per-resource circuit breaker keys** — LLM calls use `claude`, shell commands use per-step keys
4. **Test suites as golden files** — run each library's own tests, capture pass/fail, not parse test source code
5. **Structured output formats** — pytest JUnit XML, go test -json, rspec JSON, jest JSON, dotnet TRX
