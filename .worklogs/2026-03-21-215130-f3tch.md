# f3tch: concurrent file fetcher for parser development

## Summary
New standalone binary that fetches a specific file path from a list of domains. Generic tool for acquiring real-world files for any parser (security.txt, robots.txt, ads.txt, etc.).

## Features
- 5K-10K concurrent requests, no rate limiting needed (one request per domain)
- Binary file detection via `infer` crate (rejects images, archives, etc.)
- HTML/JSON/XML detection (rejects soft-404 markup responses)
- Soft-404 detection: fetches a random nonexistent path on the same domain, rejects if response is identical
- Optional `--must-contain` content filter
- Saves numbered files + manifest.json + index.json with metadata

## Test results
- 100K domains in 3.4 minutes
- 190 clean security.txt files (zero junk)
- Soft-404 detection eliminated all false positives
