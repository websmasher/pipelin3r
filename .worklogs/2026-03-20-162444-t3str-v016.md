# t3str v0.1.6: Fix Go vet, Ruby minitest, PHP legacy detection

## Summary
Three targeted fixes for the remaining runner failures, plus visibility into incompatible test suites.

## Changes

### Go: disable vet for synthesized go.mod repos
- When t3str creates go.mod (pre-modules repo), adds `-vet=off` to `go test` args
- Go 1.24's vet rejects old patterns like `t.Fatalf(err.Error())` that compiled fine on earlier versions
- Only applied to repos where we created go.mod — existing go.mod repos keep vet enabled

### Ruby: direct minitest fallback (bypasses bundler)
- After rspec and rake both fail, tries running minitest files directly with `ruby -Ilib -Itest`
- Discovers `*_test.rb` files, loads them with `LoadError`/`NameError` rescue per file
- Parses minitest summary line: "N runs, N assertions, N failures, N errors, N skips"
- Handles repos with old Gemfiles that can't install on modern Ruby

### PHP: detect and report legacy PHPUnit incompatibility
- When PHPUnit crashes on `PHPUnit_Framework_TestCase` (3.x/4.x API), returns an `Error` result with explanation instead of silent 0
- Also detects generic `Fatal error:` crashes
- Gives visibility into WHY tests didn't run rather than hiding it
