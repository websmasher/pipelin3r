# Topic filter: match file paths + case-insensitive

## Summary
Extended t3str's topic filter to match against file paths (not just function names) with case-insensitive comparison. Added `topic_keyword()` to pipeline config to derive the topic from the package name. Pipeline now passes `--topic` to `t3str extract`.

## Decisions
- **Path matching**: If a test file's path contains the topic string (case-insensitive), ALL tests in that file are relevant. This handles cases like DomainDetective where `TestSecurityTXTAnalysis.cs` has tests named `ValidSecurityTxtIsParsed` that wouldn't match a simple function-name filter.
- **Case-insensitive**: Both path and function name matching are now case-insensitive. `securitytxt` matches `SecurityTXT`, `SecurityTxt`, `security_txt`, etc.
- **Topic derivation**: `websmasher-security-txt-parser-v3` → strip prefix/suffix → `security-txt` → remove separators → `securitytxt`. This is the keyword passed to `--topic`.
- **Single-purpose libraries**: For repos like `eikendev/sectxt` where every test is relevant, the topic filter returns all tests anyway (all paths/names match).

## Key files
- `apps/t3str/crates/adapters/outbound/extract/src/helpers.rs` — path matching + case-insensitive filter
- `websmasher/tools/dev-process-v3/src/config.rs` — `topic_keyword()` method
- `websmasher/tools/dev-process-v3/src/steps/s04_clone_and_install.rs` — pass `--topic` to t3str
