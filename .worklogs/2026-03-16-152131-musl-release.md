# Add musl static binary to release workflow, bump to 0.3.0

**Date:** 2026-03-16 15:21
**Scope:** .github/workflows/release-binary.yml, apps/shedul3r/Cargo.toml

## Summary

Added x86_64-unknown-linux-musl target to the release workflow so shedul3r can run on any Linux (no glibc dependency). Bumped version to 0.3.0 to trigger a new release with MCP transport + actix-web migration + musl binary.

## Context & Problem

The glibc binary built on Ubuntu 24.04 (glibc 2.39) doesn't run on Railway's Debian Bookworm (glibc 2.36). A musl static binary has zero runtime dependencies and runs everywhere. This lets claude-worker use `cargo binstall` instead of building from source.
