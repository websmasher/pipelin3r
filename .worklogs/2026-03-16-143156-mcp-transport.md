# Add MCP transport to shedul3r

**Date:** 2026-03-16 14:31
**Scope:** apps/shedul3r/crates/adapters/inbound/mcp/ (new), rest Cargo.toml + main.rs + state.rs

## Summary

Added MCP (Model Context Protocol) inbound adapter. Exposes `execute_task` and `get_status` as MCP tools via Streamable HTTP on a separate port (REST port + 1). Uses rmcp 1.2 with built-in Axum transport for MCP, actix-web for REST.

## Decisions Made

### Separate port for MCP (not same server)
- **Chose:** MCP runs on its own Axum instance on port 7944 (REST port + 1)
- **Why:** rmcp's Streamable HTTP transport uses Axum internally. Embedding an Axum service inside actix-web would require rmcp-actix-web, which has version compatibility issues with rmcp 1.2. Two servers in one binary on different ports is simpler and clean.

### Engine wrapped in Arc
- **Chose:** Changed `AppState.engine` from `ConcreteEngine` to `Arc<ConcreteEngine>`
- **Why:** Both REST and MCP need shared access to the same engine instance
