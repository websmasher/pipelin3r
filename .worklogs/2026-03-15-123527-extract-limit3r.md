# Extract limit3r from shedul3r

**Date:** 2026-03-15 12:35
**Scope:** packages/limit3r (new), apps/shedul3r (updated)

## Summary
Extracted resilience patterns (rate limiter, circuit breaker, bulkhead, retry) from shedul3r's domain types, port traits, and db adapter into the standalone limit3r package. shedul3r now depends on limit3r via path dep. All 22 existing tests pass unchanged.

## What moved
- Config types (RateLimitConfig, etc.) → limit3r/src/config.rs
- Duration serde helper → limit3r/src/duration_serde.rs
- Trait definitions → limit3r/src/traits.rs
- In-memory implementations → limit3r/src/{rate_limiter,circuit_breaker,bulkhead,retry}.rs
- Error variants → limit3r/src/error.rs (new Limit3rError enum)

## What changed in shedul3r
- domain/types re-exports limit3r types
- SchedulrError gets Resilience(#[from] Limit3rError) variant
- ports/outbound/repo re-exports limit3r traits
- db adapter imports from limit3r instead of domain_types/repo

## Key Files for Context
- packages/limit3r/src/lib.rs — crate root with all re-exports
- apps/shedul3r/crates/domain/types/src/error.rs — wrapping variant
