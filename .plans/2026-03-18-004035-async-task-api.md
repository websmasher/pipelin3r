# Async task API for shedul3r

**Date:** 2026-03-18 00:40
**Task:** Add async task submission + polling to shedul3r, SDK, and pipelin3r.

## Changes

### shedul3r (server)
- Add `AsyncTaskStore` to domain types: `BTreeMap<String, AsyncTaskState>` behind `Arc<RwLock<>>`
- `AsyncTaskState`: `Running` | `Completed(TaskResponse)` | `Failed(String)`, with `created_at` for TTL cleanup
- Add to AppState
- New endpoints:
  - `POST /api/tasks/async` → 202 `{ task_id }` (spawns task in background)
  - `GET /api/tasks/{id}` → `{ status: "running" | "completed" | "failed", result?: TaskResponse }`
- Background reaper: clean completed tasks after 10 min TTL
- Existing `POST /api/tasks` (sync) stays unchanged

### shedul3r-rs-sdk
- Add `submit_task_async(&TaskPayload) -> Result<String, SdkError>` (returns task_id)
- Add `get_task_status(&str) -> Result<AsyncTaskStatus, SdkError>`
- Add `submit_task_poll(&TaskPayload) -> Result<TaskResult, SdkError>` (async submit + poll loop)

### pipelin3r
- `execute_with_work_dir`: when remote, use `submit_task_poll` instead of `submit_task`
