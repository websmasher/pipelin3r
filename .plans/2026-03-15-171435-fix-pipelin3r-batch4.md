# Fix pipelin3r Batch 4 issues

**Date:** 2026-03-15 17:14
**Task:** Fix 7 issues from adversarial review v2, Batch 4

## Goal
Fix all 7 identified issues in pipelin3r: path traversal in bundle, template cross-injection, Tool enum, dry-run meta enrichment, error variant expansion, template ergonomics, executor convenience methods.

## Approach

### 1. Bundle path traversal (HIGH)
- Add `validate_path()` to bundle.rs that rejects any non-Normal path components
- Call it in `add_file` and `add_text_file` (change return type to Result)
- Update tests

### 2. Template cross-injection (HIGH)
- Combine simple + content replacements into one single-pass operation in `fill()`
- Remove sequential phase-1 replacement
- Update tests to verify simple-to-simple injection is prevented

### 3. Tool enum (MEDIUM)
- Add Tool enum to model.rs (alongside Model/Provider)
- Update AgentBuilder::tools() to accept &[Tool]
- Update task.rs to format from Tool
- Export from lib.rs

### 4. Dry-run meta enrichment (MEDIUM)
- Add environment key names and bundle file paths to meta.json in dry-run capture

### 5. PipelineError variants (MEDIUM)
- Add AgentFailed, BatchPartialFailure, Timeout variants
- Update agent.rs require_success to use AgentFailed
- Update pool error paths

### 6. TemplateFiller ergonomics (MEDIUM)
- Change set/set_content to take `mut self` and return `Self`
- Add Template::from_file
- Update all call sites and tests

### 7. Executor convenience methods (LOW)
- Add command() and transform() to Executor
