## Phase 4: Pipeline Integration

### Goal

Replace the current pipeline's Python-script-generation approach with `t3str extract` and `t3str run` calls. The pipeline currently:

1. Generates Python scripts that use regex to find tests (`s04_clone_and_install.rs`, phase 4c)
2. Executes those scripts and parses stdout with regex (`s06_run_parsers.rs`)

After integration, it will:

1. Call `t3str extract --language <lang> --repo-dir <path> --format json` to discover tests
2. Call `t3str run --language <lang> --repo-dir <path> --format json` to execute tests
3. Parse the structured JSON output directly (no regex)

### Files to Modify

| File | What Changes |
|------|-------------|
| `packages/pipelin3r/src/pipeline.rs` | Add t3str binary path to pipeline config |
| `packages/pipelin3r/src/executor/mod.rs` | May need new execution mode for t3str |
| `packages/pipelin3r/src/bundle_dir.rs` | Ensure t3str binary is bundled/available |
| `packages/pipelin3r/src/template/mod.rs` | Remove Python script template generation for test discovery |
| `packages/pipelin3r/src/transform/mod.rs` | Replace regex parsing with JSON deserialization of t3str output |

### Key Design Decisions for Phase 4

1. **t3str must be a pre-built binary** bundled alongside the pipeline, not compiled on-the-fly. The pipeline runs in isolated environments — it cannot `cargo build` t3str.

2. **JSON is the integration format.** Both `t3str extract` and `t3str run` support `--format json`. The pipeline should always use JSON mode and deserialize into the existing domain types.

3. **Error handling**: t3str uses exit codes + structured error JSON. The pipeline should treat non-zero exit as a test failure (not a pipeline crash) and include the error output in results.

4. **Topic filtering**: `t3str extract` supports `--topic <filter>` for substring matching on test names. The pipeline should pass this through from the pipeline step config.

5. **No Python anywhere.** The entire point of t3str is eliminating generated Python scripts. Phase 4 must remove all Python script generation, not just add t3str as an alternative path.

### How to Build and Test t3str

```bash
# Build
cd apps/t3str && cargo build --release

# Run tests
cd apps/t3str && cargo test --workspace

# Lint
cd apps/t3str && cargo fmt --all -- --check
cd apps/t3str && cargo clippy --workspace --all-targets -- -D warnings

# Live test (extract)
./apps/t3str/target/release/t3str extract --language rust --repo-dir . --format json

# Live test (run)
./apps/t3str/target/release/t3str run --language rust --repo-dir apps/t3str --format json
```
