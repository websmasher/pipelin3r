# Implement t3str CLI adapter crate

**Date:** 2026-03-20 09:45
**Task:** Create the CLI binary crate (composition root) for t3str

## Goal
Replace the stub `lib.rs` in `apps/t3str/crates/adapters/inbound/cli/` with a proper `main.rs` binary that wires adapters to commands via clap CLI.

## Approach

### Step-by-step plan
1. Add `serde_json` and `t3str-discovery-port` deps to Cargo.toml
2. Delete `src/lib.rs`, create `src/main.rs` with clap CLI definition
3. Handle Language parsing via a wrapper type (Language doesn't derive ValueEnum)
4. Wire Extract and Run commands to their adapter implementations
5. Handle clippy strictness: no unwrap/expect, no bare print, no process::exit, use ExitCode
6. Create `clippy.toml` with appropriate bans for composition root layer
7. Test compilation

### Key decisions
- **Language wrapper**: Since Language in domain-types doesn't derive clap::ValueEnum, accept as String and parse via match statement
- **ExitCode over process::exit**: process::exit is banned in clippy.toml, use std::process::ExitCode
- **Manual tokio runtime**: Can't use #[tokio::main] with ExitCode return cleanly, build runtime manually
- **Allow print_stdout/print_stderr**: CLI output is the primary interface, needs item-level #[allow] with justification
