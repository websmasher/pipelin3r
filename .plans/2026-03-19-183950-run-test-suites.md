# Rewrite step 6: Run each library's own test suite, capture golden results

**Date:** 2026-03-19 18:39
**Task:** Run test suites remotely, capture structured pass/fail per test, build golden map

## Approach

For each cloned library, run its test suite using the language's native test runner with structured output. Parse the results into a golden map.

### Per-language test commands

| Language | Command | Output format |
|----------|---------|---------------|
| Python | `pytest --tb=short -q 2>&1` or `python -m pytest -v` | Parse verbose output |
| Go | `go test -v -json ./...` | JSON lines, one per test event |
| Rust | `cargo test -- --format=json -Z unstable-options` or `cargo test 2>&1` | Parse verbose |
| PHP | `vendor/bin/phpunit --log-junit results.xml` or `vendor/bin/phpunit 2>&1` | JUnit XML or parse verbose |
| Ruby | `bundle exec rspec --format json` or `bundle exec rake test 2>&1` | JSON or parse verbose |
| JavaScript | `npx jest --json` or `npm test 2>&1` | JSON or parse verbose |
| C# | `dotnet test --verbosity normal 2>&1` | Parse verbose |

### Output structure

Each library produces a results file: `ground-truth/{lang}-{lib}/test-results.json`

```json
{
  "library": "python-digitaltrustcenter-sectxt",
  "language": "python",
  "total": 31,
  "passed": 28,
  "failed": 2,
  "skipped": 1,
  "tests": [
    { "name": "test_parse_contact", "status": "pass" },
    { "name": "test_invalid_expires", "status": "fail", "message": "..." },
    { "name": "test_skip_reason", "status": "skip" }
  ]
}
```

### Implementation

Step 6 becomes: for each library, submit a remote command that:
1. cd to the clone directory
2. Run the test suite with the appropriate command
3. Write structured results to a file in the work_dir
4. Download via expect_outputs

Use a Python script on the remote side to run tests and parse output into JSON (avoids shell escaping issues). One remote command per library, batched with concurrency.
