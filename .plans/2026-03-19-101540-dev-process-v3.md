# Dev-process V3: First 3 steps on new verified primitives

**Date:** 2026-03-19 10:15
**Task:** Port dev-process-v2 steps 1-3 to use PromptedStep/VerifiedStep/run_verified_step

## Goal
Create `tools/dev-process-v3/` in the websmasher repo with the first 3 pipeline steps reimplemented using the new pipelin3r verified step primitives. Compare the API ergonomics and output with V2.

## Key Differences: V2 → V3

| Aspect | V2 | V3 |
|--------|----|----|
| Prompt loading | `include_str!()` + inline `TemplateFiller` | `PromptedStep` with `prompt_template` path + `Var` |
| Template vars | Manual `TemplateFiller::new().set().fill()` | `Var::String` and `Var::File` in the step definition |
| Verification | None — doer runs once, hope for the best | Optional `VerifiedStep` with breaker/fixer loop |
| File routing | Manual `AgentStep { config, inputs, outputs }` | `PromptedStep { prompt_template, vars, inputs, outputs }` auto-resolves to `AgentStep` |
| Error handling | Manual `match result { Ok(r) if r.success => ... }` | `require_success()` + `require_converged()` |

## Approach

### Step 1: Scaffold (VerifiedStep with format breaker)
- **Doer**: Same prompt as V2, produces `architecture.md`
- **Breaker**: Script breaker that checks `architecture.md` has required sections (# header, ## Spec reference, ## Boundaries, etc.)
- **Fixer**: Prompt that gets the issues and current architecture.md, fixes missing sections
- **Max iterations**: 2

### Step 2: Research libraries (batch PromptedStep, no verification)
- Same as V2 but using `PromptedStep` + `Var::File` for architecture.md
- 9 parallel agents via `PipelineContext::run_agent_batch`
- No verification needed — research is exploratory

### Step 3: Extract libraries JSON (batch PromptedStep with JSON validation)
- Uses `PromptedStep` + `Var::File` for overview.md content
- Could optionally add JSON validation breaker but keeping it simple for comparison
