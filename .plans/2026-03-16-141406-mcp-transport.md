# Add MCP transport to shedul3r

**Date:** 2026-03-16 14:14
**Task:** Add MCP (Model Context Protocol) inbound adapter to shedul3r so Claude Code can call it directly as an MCP tool.

## Goal

Claude Code can connect to shedul3r via: `claude mcp add --transport http shedul3r https://host/mcp`
And then call tools like `execute_task`, `get_status` directly without curl/scripts.

## Approach

### Architecture: MCP scope on the same server

Mount the MCP Streamable HTTP endpoint at `/mcp` on the same actix-web server. Both REST routes and MCP share the same `AppState` and port.

### Dependencies
- `rmcp` 0.16+ with features: `server`, `macros`
- `rmcp-actix-web` 0.12+ for Streamable HTTP transport
- `schemars` 0.8 for JSON schema generation from tool parameters

### MCP tools to expose

1. **`execute_task`** — Run a task through the engine
   - Params: task (YAML string), input (prompt), working_directory, environment (JSON object), timeout_ms, limiter_key
   - Returns: success, output, exit_code, elapsed

2. **`get_status`** — Scheduler status
   - No params
   - Returns: active_tasks, pending_tasks, started_at

### Implementation

1. Create `crates/adapters/inbound/mcp/` crate (library, not binary)
2. Define `McpServer` struct holding `Arc<AppState>`
3. Implement tools with `#[tool_router]` / `#[tool]` macros
4. Implement `ServerHandler` with `#[tool_handler]`
5. Export a function that creates the `StreamableHttpService`
6. In `rest/src/main.rs`: import mcp crate, mount at `/mcp` scope

### Key decisions
- **Library crate, not binary** — the REST crate's main.rs wires everything
- **StreamableHttpService created OUTSIDE HttpServer::new()** — required for shared session manager across workers
- **Same port** — one HTTP port serves both REST and MCP
- **Auth** — MCP endpoint uses the same SHEDUL3R_API_KEY via actix middleware
