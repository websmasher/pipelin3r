# Implement image_gen module for pipelin3r

**Date:** 2026-03-17 19:54
**Task:** Create the image generation module following the config-struct API v2 design.

## Goal
Add `src/image_gen/` module with types, HTTP client, and `generate_image()` standalone function. Uses `ImageGenHttpConfig` (shared HTTP/auth/rate-limit state) and `ImageGenConfig` (per-call params). Does NOT go through Executor -- standalone function calling OpenRouter directly.

## Approach

### Files to create:
1. `src/image_gen/types.rs` -- AspectRatio, ImageModel, RefImage, RefImageRole, serde request/response structs
2. `src/image_gen/client.rs` -- generate_image_request(), parse_image_response(), fetch_cost()
3. `src/image_gen/mod.rs` -- ImageGenHttpConfig, ImageGenConfig, ImageGenResult, generate_image()
4. `src/image_gen/tests.rs` -- unit tests for types and request/response parsing

### Files to modify:
- `Cargo.toml` -- add reqwest, serde, limit3r, base64
- `src/error.rs` -- add ImageGenFailed variant
- `src/lib.rs` -- add pub mod image_gen and re-exports

### Key decisions:
- Config-struct API (not builder) per v2 plan
- Standalone `generate_image()` function, not Executor method
- ImageGenHttpConfig holds reqwest::Client, api_key, optional rate limiter
- Rate limiting via limit3r::InMemoryRateLimiter + RateLimitConfig
- No unsafe, no unwrap, no HashMap, doc comments on all pub items
