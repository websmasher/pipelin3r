# Fix 7 bugs/design issues in pipelin3r crate

**Date:** 2026-03-17 16:21
**Task:** Fix bugs 3-9 in agent execution code

## Goal
Fix all 7 identified bugs and design issues in the pipelin3r crate's agent execution code.

## Approach

### Bug 3: Batch dry-run skips validate_work_dir
- In `execute_batch_task_dry_run`, add `validate_work_dir` call at the start if `task.work_dir` is Some.

### Bug 4: Dry-run counter is global not per-agent
- Change `DryRunConfig.counter: usize` to `counters: BTreeMap<String, usize>` in executor/mod.rs.
- Update `execute_dry_run_capture` to look up per-step counter.

### Bug 5: Batch dry-run error propagation differs from real mode
- In `AgentBatchBuilder::execute()` dry-run branch, catch errors per-item instead of propagating with `?`.

### Design issue 6: Root `/` passes validate_work_dir
- Add check rejecting root path in `validate_work_dir`.

### Design issue 7: Empty subdirectories dropped during remote upload
- Add documentation comment only.

### Design issue 8: Symlink directory loops
- Add `BTreeSet<PathBuf>` visited tracking to `read_dir_to_memory_inner` and `collect_relative_paths_inner`.

### Design issue 9: Batch duplicate-dir check uses string comparison
- Canonicalize paths before inserting into the BTreeSet in the duplicate check.
