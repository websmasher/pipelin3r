# Phase 3: t3str Extract Adapter — Tree-Sitter Test Discovery

## Goal

Replace all 9 stub `discover()` functions in the extract adapter with working tree-sitter-based test discovery. Each language module walks the repo for test files, parses them with tree-sitter, extracts test function names via queries, applies topic filters, and returns `Vec<TestFile>`.

## Approach

### Dependencies

Add to workspace `Cargo.toml`:
```toml
tree-sitter = "0.24"
streaming-iterator = "0.1"    # Required for QueryMatches in ts 0.24
tree-sitter-rust = "0.23.2"
tree-sitter-python = "0.23.6"
tree-sitter-go = "0.23.4"
tree-sitter-javascript = "0.23.1"
tree-sitter-php = "0.24.2"
tree-sitter-c-sharp = "0.23.1"
tree-sitter-ruby = "0.23.1"
tree-sitter-java = "0.23.5"
tree-sitter-elixir = "0.3.5"
```

Add to extract adapter `Cargo.toml`: all of the above as workspace dependencies.

### Shared Infrastructure — `helpers.rs`

Two shared functions:

1. **`find_test_files(repo_dir, extensions, is_test_file_fn)`** — Walks directories recursively, skips common non-source dirs (`.git`, `node_modules`, `vendor`, `target`, `__pycache__`, `.venv`, `build`, `dist`), filters by extension + naming convention.

2. **`extract_with_query(source, ts_language, query_str)`** — Parses source with tree-sitter, runs query, collects captured function names. Uses `streaming_iterator::StreamingIterator` for QueryMatches.

### Per-Language Implementation

Each module:
1. Defines file extensions and test file predicate
2. Defines tree-sitter query string for test function discovery
3. Calls helpers to find files and extract test names
4. Applies topic filter (name contains filter string)
5. Returns `Vec<TestFile>`

#### Language-Specific Test Patterns

| Language | Test File Pattern | Test Function Pattern |
|----------|------------------|----------------------|
| Rust | `*.rs` in `tests/` or containing `#[test]` | `#[test]` attribute on `fn` |
| Python | `test_*.py`, `*_test.py` | `def test_*()` functions |
| Go | `*_test.go` | `func Test*(t *testing.T)` |
| JavaScript | `*.test.js`, `*.spec.js`, `test/*.js` | `it()`, `test()` calls |
| PHP | `*Test.php`, `test*.phpt` | `function test*()` methods |
| C# | `*Test.cs`, `*Tests.cs` | Methods with `[Test]`, `[Fact]`, `[TestMethod]` |
| Ruby | `*_spec.rb`, `test_*.rb` | `it` blocks, `def test_*` methods |
| Java | `*Test.java`, `Test*.java` | Methods with `@Test` annotation |
| Elixir | `*_test.exs` in `test/` | `test "name"` blocks |

### Key Decisions

- **tree-sitter queries over manual AST walking**: Queries are declarative, concise, and handle nesting correctly. The `#match?` predicate in queries uses tree-sitter's internal regex (allowed as transitive dep).
- **Name-based topic filtering**: `topic_filter` is a simple substring match on test function names. This is consistent with how the run adapter uses `--filter`.
- **Skip directories, not individual files**: Walking skips known non-source directories at the directory level for performance, rather than reading every file.
