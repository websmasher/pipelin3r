# Implement JUnit XML Parser for t3str run adapter

**Date:** 2026-03-20 10:02
**Task:** Create JUnit XML parser in `crates/adapters/outbound/run/src/parsers/junit_xml.rs`

## Goal
Parse JUnit XML format (used by pytest, phpunit, maven surefire) into `Vec<TestResult>`. Handle both `<testsuites>` and bare `<testsuite>` root elements.

## Approach

### Step-by-step plan
1. Create `crates/adapters/outbound/run/src/parsers/junit_xml.rs` with `parse(xml: &str) -> Result<Vec<TestResult>, T3strError>`
2. Use `quick_xml::Reader` event-based parsing to iterate XML events
3. On `<testcase>` start: extract classname, name, time attributes
4. Read child events until `</testcase>`: detect failure/error/skipped children
5. Build `TestResult` from collected data
6. Register module in `parsers/mod.rs`
7. Add comprehensive tests

### Key decisions
- **Event-based parsing over serde deserialization:** More control over edge cases (missing attributes, both root forms), no need for intermediate structs.
- **Duration conversion:** Use `f64::round()` then convert via `u64::try_from` or just `as` with a safe cast pattern. Actually, `round()` returns f64, need to convert to u64 without `as`. Will use `#[allow(clippy::cast_possible_truncation, clippy::cast_sign_loss)]` with justification since we know the value is non-negative and within u64 range after multiplying seconds by 1000.
