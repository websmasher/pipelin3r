# Implement three JSON-based parsers for t3str run adapter

**Date:** 2026-03-20 10:02
**Task:** Create go_json, jest_json, and rspec_json parsers in the run adapter's parsers module.

## Goal
Three new parser modules that convert Go test JSON, Jest JSON, and RSpec JSON output into unified `TestResult` format. All must pass strict clippy lints.

## Approach

### Step-by-step plan
1. Add shared `seconds_to_ms` helper to `parsers/mod.rs` with appropriate `#[allow]` for `as_conversions`
2. Create `go_json.rs` ‚Äî NDJSON line-by-line parser with dedup via BTreeSet
3. Create `jest_json.rs` ‚Äî nested JSON parser supporting both `testResults` and `assertionResults` arrays
4. Create `rspec_json.rs` ‚Äî flat JSON parser for `examples` array
5. Register all three modules in `parsers/mod.rs`
6. Run `cargo test` and `cargo clippy` to verify

### Key decisions
- **Shared `seconds_to_ms`:** Avoids duplicating float‚Üíu64 conversion logic with its `#[allow]` annotations
- **`mul_add` for arithmetic:** Avoids `arithmetic_side_effects` lint on `seconds * 1000.0`
- **`serde_json::Value` for Go JSON:** Lines may have varying shapes, Value-based parsing is more robust than typed deserialization for NDJSON with mixed content
