## Command Reference

### Rust

```
guardrail3 rs init [PATH] --profile <service|library> [--force]
```
Creates `guardrail3.toml`, `local/` override directory, and release configs (service only).

```
guardrail3 rs generate [PATH]
```
Reads `guardrail3.toml` → produces `clippy.toml`, `deny.toml`, `rustfmt.toml`, `rust-toolchain.toml`, per-crate `clippy.toml`, `.githooks/pre-commit`.

```
guardrail3 rs validate [PATH] [OPTIONS]
  --format <text|json|md>       Output format (default: text)
  --staged                      Only git-staged files
  --dirty                       Staged + unstaged files
  --commits <N>                 Files changed in last N commits
  --files <FILE...>             Specific files only
  --code                        Only code quality checks
  --architecture                Only architecture checks
  --release                     Only release readiness checks
  --tests                       Only test quality checks
  --thorough                    Include slow checks (cargo publish --dry-run)
  --verbose                     Show every audit trail item individually
```

```
guardrail3 rs check [PATH]           # CI: verify configs not stale
guardrail3 rs diff [PATH]            # Dry-run of rs generate
guardrail3 rs hooks-install [PATH]   # Regenerate pre-commit hook only
guardrail3 rs hooks-validate [PATH]  # Check hook setup
guardrail3 rs list-modules           # List embedded config modules
guardrail3 rs show-module <NAME>     # Print module content
```

## Profiles (Rust only — TypeScript has no profiles)

| Profile | Use for | What it bans |
|---------|---------|-------------|
| `service` | HTTP servers, CLI tools, binaries | `std::fs` (must centralize), `process::exit`, env mutation, `HashMap` (use `BTreeMap`), `Mutex` (use `parking_lot`). Allows I/O crates in adapter layer. Composition-root crate may use `LazyLock`. |
| `library` | Shared packages, pure logic | Everything in service PLUS bans ALL I/O crates (`axum`, `tokio`, `reqwest`, `sqlx`, `hyper`, `diesel`) and ALL global state (`LazyLock`, `OnceLock`, `once_cell`). MUST have `allowed_deps`. |

## Architecture Convention

### Rust services (`apps/`)

Services MUST use hex arch with separate crates:

```
apps/my-api/
  crates/
    domain/           Pure types, zero deps on ports/adapters
    ports/
      inbound/        Traits for incoming requests
      outbound/       Traits for external services
    app/              Business logic, uses ports via trait bounds
    adapters/
      inbound/api/    HTTP handlers (axum, etc.)
      outbound/       DB, HTTP clients, filesystem impls
```

**Dependency flow (enforced by R-ARCH-02):**

| Layer | Can depend on | Cannot depend on |
|-------|--------------|-----------------|
| `domain` | nothing | ports, app, adapters |
| `ports` | domain | app, adapters |
| `app` | domain, ports | adapters |
| `adapters` | everything | — |

### Rust libraries (`packages/`)

```
packages/my-lib/          Pure logic, allowed_deps enforced
packages/my-sdk/          May need network — explicit in allowed_deps
```

Libraries MUST have `allowed_deps` in `guardrail3.toml` listing every permitted dependency.

### TypeScript services (`apps/`)

```
apps/my-app/src/
  app/                    Next.js router (reserved name)
  components/             UI components
  lib/                    Shared utilities
  modules/                Hex arch lives here
    domain/               Pure types
    ports/
      inbound/            Route/API interfaces
      outbound/           Repository/client interfaces
    application/          Use cases (NOT "app" — Next.js reserves it)
      commands/
    adapters/
      inbound/api/        API route handlers
      outbound/           DB clients, HTTP clients
```

**Import boundary rules (enforced by T-ARCH-02):**

| Layer | Can import from | Cannot import from |
|-------|----------------|-------------------|
| `domain` | nothing in modules/ | ports, application, adapters |
| `ports` | domain | application, adapters |
| `application` | domain, ports | adapters |
| `adapters` | everything | — |

### TypeScript packages

```
packages/my-lib/          Shared logic (TypeScript library)
```

## Config File (`guardrail3.toml`)

Only needed for `generate`/`check`/`diff`. Not needed for `validate`.

### Single Rust service

```toml
[profile]
name = "service"

[rust]
workspace_root = "."
```

### Single Rust library

```toml
[profile]
name = "library"

[rust]
workspace_root = "."
```

### Workspace / monorepo

```toml
[profile]
name = "service"

[rust]
workspace_root = "."

# --- Services (in apps/) ---
[rust.crates.my-api]
profile = "service"
layer = "composition-root"

# --- Libraries (in packages/) ---
[rust.crates.my-lib]
profile = "library"
layer = "pure"
allowed_deps = ["serde", "thiserror", "chrono"]

[rust.crates.my-sdk]
profile = "library"
allowed_deps = ["serde", "serde_json", "reqwest", "tokio", "thiserror"]
```

### Per-crate fields

| Field | Values | Description |
|-------|--------|------------|
| `profile` | `"service"` or `"library"` | Overrides workspace default for this crate |
| `layer` | `"composition-root"` or `"pure"` | `composition-root` allows `LazyLock`; `pure` bans global state |
| `allowed_deps` | `["serde", "thiserror", ...]` | Dependency allowlist. Any `[dependencies]` entry NOT listed = R-DEPS-01 error. `[dev-dependencies]` and `[build-dependencies]` not checked. Workspace path deps not checked. |

### local/ overrides (created by `rs init`)

| File | Purpose |
|------|---------|
| `local/clippy-methods.toml` | Extra disallowed methods |
| `local/clippy-types.toml` | Extra disallowed types |
| `local/deny-bans.toml` | Extra crate bans |
| `local/deny-skip.toml` | Duplicate crate skip entries |
| `local/deny-feature-bans.toml` | Feature bans |

## Setup Guide

### Step 1: Identify project type and run init

| Project type | Command |
|-------------|---------|
| Single Rust service | `guardrail3 rs init --profile service .` |
| Single Rust library | `guardrail3 rs init --profile library .` |
| Rust workspace | `guardrail3 rs init --profile service .` then configure per-crate |
| TypeScript project | `guardrail3 ts init .` |
| Rust + TypeScript monorepo | `guardrail3 rs init --profile service .` then `guardrail3 ts init .` |

### Step 2: Configure per-crate settings (workspaces only)

Read `Cargo.toml` `[workspace.members]`. For each crate:
- Crates in `apps/` → `profile = "service"`, `layer = "composition-root"`
- Crates in `packages/` → `profile = "library"`, `layer = "pure"`, `allowed_deps = [...]`

For `allowed_deps`: read each crate's `Cargo.toml` `[dependencies]` and list every dependency. Anything not listed = R-DEPS-01 error.

### Step 3: Generate

```
guardrail3 rs generate    # Rust configs + hooks
guardrail3 ts generate    # TypeScript configs + hooks
```

### Step 4: Validate

```
guardrail3 rs validate .  # Rust checks
guardrail3 ts validate .  # TypeScript checks
```

Fix errors, re-run until clean. Exit code 1 = errors found.

### Step 5: CI

```
guardrail3 rs check       # Fails if generated configs are stale
```
